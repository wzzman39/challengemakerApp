# ChallengeMaker

## Запуск локально

1. `npm install`
2. Скопируй `.env.example` → `.env.local` (уже с твоими реальными URL/key, менять не нужно)
3. `npm run dev` → открой ссылку из терминала (обычно http://localhost:5173)
4. Зарегистрируйся email+паролем (кнопка "Нет аккаунта? Создать")

## Важно про подтверждение email

Supabase по умолчанию требует подтверждения почты письмом. Если после регистрации
пишет "проверь почту" — либо подтверди по ссылке из письма, либо в Supabase:
**Authentication → Providers → Email → выключи "Confirm email"** (для личного
проекта это нормально, письма никто кроме тебя всё равно не увидит).

## Деплой на телефон (Vercel)

1. Залей папку в GitHub-репозиторий (`git init`, `git add .`, `git commit`, пуш).
   `.env.local` в репозиторий не попадёт — он в `.gitignore`.
2. На vercel.com → **New Project** → импортируй репозиторий.
3. В настройках проекта → **Environment Variables** добавь:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
   (значения — из `.env.example`)
4. Deploy. Vercel даст ссылку вида `challengemaker.vercel.app`.
5. На телефоне открой ссылку в браузере → меню → **"Добавить на экран домой"**.

Дальше любой `git push` в основную ветку — автообновление на телефоне
(достаточно переоткрыть приложение).

## Структура

```
src/
  App.jsx          — экраны, обработчики действий
  styles.css        — вся стилизация
  lib/
    supabaseClient.js — подключение к Supabase
    db.js              — все запросы к БД в одном месте
    seed.js            — стартовые данные при первом входе
    constants.js       — баланс (CHAOS/EXP), метаданные статов
    utils.js           — даты, уровни, хелперы
```
